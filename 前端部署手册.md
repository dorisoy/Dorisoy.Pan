# Dorisoy.Pan 前端（Angular）部署手册

本手册介绍如何在 Linux 服务器上部署 Dorisoy.Pan 前端 Angular 应用，并与已运行的后端 API 配合使用。

---

## 一、当前后端环境

后端 API 已通过 Docker 运行，端口映射如下：

```
容器名称: dorisoy.pan
HTTP:  0.0.0.0:5000 -> 80
HTTPS: 0.0.0.0:5001 -> 443
```

即后端 API 地址为：`http://服务器IP:5000`

---

## 二、环境要求

| 组件 | 版本要求 |
|---|---|
| Node.js | 14.x ~ 16.x（Angular 12 兼容范围） |
| npm | 6.x+ |
| Nginx | 1.18+（用于托管静态资源和反向代理） |

> **重要**：Angular 12 不兼容 Node.js 18+，请使用 Node.js 14 或 16。

---

## 三、构建前端

### 3.1 安装 Node.js 16（推荐）

```bash
# 使用 NodeSource 安装 Node.js 16
curl -fsSL https://deb.nodesource.com/setup_16.x | sudo -E bash -
sudo apt install -y nodejs

# 验证
node -v   # 应输出 v16.x.x
npm -v    # 应输出 8.x.x
```

如果已有更高版本 Node.js，建议使用 nvm 管理多版本：

```bash
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.7/install.sh | bash
source ~/.bashrc
nvm install 16
nvm use 16
```

### 3.2 安装依赖

```bash
cd /path/to/source/UI
npm install
```

> 如果 `npm install` 过程中出现 OpenSSL 相关错误，执行：
> ```bash
> export NODE_OPTIONS=--openssl-legacy-provider
> ```

### 3.3 修改 API 地址（关键步骤）

编辑生产环境配置文件 `src/environments/environment.prod.ts`：

```typescript
export const environment = {
  production: true,
  apiUrl: '/',    // 方式A：同域部署，通过 Nginx 反向代理（推荐）
  // apiUrl: 'http://你的服务器IP:5000/',  // 方式B：跨域直连后端
  officeDocumentExtensions: ['.doc', '.docx', '.ppt', '.pptx', '.xls', '.xlsx'],
  imageExtensions: ['.jpg', '.jpeg', '.png', '.gif', '.tiff', '.psd', '.bmp',
    '.webp', '.raw', '.bmp', '.heif', '.indd', '.svg', '.ai', '.eps'],
  audioFileExtension: ['.3gp', '.aa', '.aac', '.aax', '.act', '.aiff', '.alac', '.amr',
    '.ape', '.au', '.awb', '.dss', '.dvf', '.flac', '.gsm', '.iklx', '.ivs', '.m4a',
    '.m4b', '.m4p', '.mmf', '.mp3', '.mpc', '.msv', '.nmf', '.ogg', '.oga', '.mogg',
    '.opus', '.org', '.ra', '.rm', '.raw', '.rf64', '.sln', '.tta', '.voc', '.vox',
    '.wav', '.wma', '.wv', '.webm'],
  videoFileExtension: ['.webm', '.flv', '.vob', '.ogv', '.ogg', '.drc', '.avi', '.mts',
    '.m2ts', '.wmv', '.yuv', '.viv', '.mp4', '.m4p', '.3pg', '.flv', '.f4v', '.f4a']
};
```

**两种 API 连接方式说明：**

| 方式 | `apiUrl` 值 | 适用场景 |
|---|---|---|
| A. 同域反向代理（推荐） | `'/'` | Nginx 同时托管前端静态文件并代理 `/api` 到后端 |
| B. 跨域直连 | `'http://服务器IP:5000/'` | 前后端分开部署，后端已配置 CORS |

### 3.4 执行生产构建

```bash
export NODE_OPTIONS=--openssl-legacy-provider
npx ng build --configuration production
```

构建产物输出到 `dist/your-drive/` 目录。

### 3.5 上传构建产物到服务器

```bash
# 在服务器上创建目录
sudo mkdir -p /opt/dorisoy-pan/www

# 将构建产物上传到服务器（从开发机器执行）
scp -r dist/your-drive/* user@你的服务器IP:/opt/dorisoy-pan/www/

# 设置权限
sudo chown -R www-data:www-data /opt/dorisoy-pan/www
```

## 四、Nginx 部署配置

### 4.1 安装 Nginx

```bash
sudo apt update
sudo apt install -y nginx
sudo systemctl enable nginx
```

### 4.2 方式 A：同域部署（推荐）

前端和 API 使用同一域名，Nginx 托管前端静态文件并反向代理 API 请求到后端容器。

创建 `/etc/nginx/sites-available/dorisoy-pan`：

```nginx
server {
    listen 80;
    server_name your-domain.com;   # 替换为你的域名或 IP

    # 前端静态文件
    root /opt/dorisoy-pan/www;
    index index.html;

    # Angular 路由支持（SPA 单页应用）
    location / {
        try_files $uri $uri/ /index.html;
    }

    # API 反向代理到后端 Docker 容器
    location /api/ {
        proxy_pass http://127.0.0.1:5000/api/;
        proxy_http_version 1.1;

        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;

        # 大文件上传支持
        client_max_body_size 500M;
        proxy_read_timeout 600s;
        proxy_send_timeout 600s;
    }

    # SignalR WebSocket 代理
    location /userHub {
        proxy_pass http://127.0.0.1:5000/userHub;
        proxy_http_version 1.1;

        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;

        proxy_read_timeout 86400s;
    }

    # Swagger 文档代理（可选，仅调试用）
    location /swagger {
        proxy_pass http://127.0.0.1:5000/swagger;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }

    # 静态资源缓存
    location ~* \.(js|css|png|jpg|jpeg|gif|ico|svg|woff|woff2|ttf|eot)$ {
        expires 30d;
        add_header Cache-Control "public, immutable";
    }

    # Gzip 压缩
    gzip on;
    gzip_types text/plain text/css application/json application/javascript text/xml application/xml application/xml+rss text/javascript;
    gzip_min_length 1000;
}
```

### 4.3 方式 B：独立部署（前后端分离）

如果前端与后端在不同服务器或不同端口，前端仅托管静态文件，API 请求直连后端。

此方式需要将 `environment.prod.ts` 中 `apiUrl` 设为后端完整地址：

```typescript
apiUrl: 'http://后端服务器IP:5000/',
```

Nginx 配置 `/etc/nginx/sites-available/dorisoy-pan`：

```nginx
server {
    listen 80;
    server_name your-domain.com;

    root /opt/dorisoy-pan/www;
    index index.html;

    # Angular SPA 路由
    location / {
        try_files $uri $uri/ /index.html;
    }

    # 静态资源缓存
    location ~* \.(js|css|png|jpg|jpeg|gif|ico|svg|woff|woff2|ttf|eot)$ {
        expires 30d;
        add_header Cache-Control "public, immutable";
    }

    gzip on;
    gzip_types text/plain text/css application/json application/javascript text/xml;
    gzip_min_length 1000;
}
```

> **注意**：此方式需要后端 API 的 `appsettings.json` 中 `WebApplicationUrl` 配置包含前端地址，以允许 CORS 跨域请求。

### 4.4 启用站点

```bash
# 删除默认站点（可选）
sudo rm -f /etc/nginx/sites-enabled/default

# 启用配置
sudo ln -s /etc/nginx/sites-available/dorisoy-pan /etc/nginx/sites-enabled/

# 测试配置语法
sudo nginx -t

# 重载 Nginx
sudo systemctl reload nginx
```

### 4.5 配置 HTTPS（推荐）

```bash
sudo apt install -y certbot python3-certbot-nginx
sudo certbot --nginx -d your-domain.com
```

证书自动续期。启用 HTTPS 后注意同步修改后端 `appsettings.json` 中的 `WebApplicationUrl`。

## 五、Docker 方式部署前端（可选）

如果更倾向于使用 Docker 部署前端，可创建前端 Dockerfile。

### 5.1 创建 Dockerfile

在 `UI/` 目录下创建 `Dockerfile`：

```dockerfile
# 构建阶段
FROM node:16-alpine AS build
WORKDIR /app

COPY package.json package-lock.json ./
RUN npm install

COPY . .
ENV NODE_OPTIONS=--openssl-legacy-provider
RUN npx ng build --configuration production

# 运行阶段
FROM nginx:alpine
COPY --from=build /app/dist/your-drive /usr/share/nginx/html
COPY nginx.conf /etc/nginx/conf.d/default.conf
EXPOSE 80
```

### 5.2 创建 nginx.conf

在 `UI/` 目录下创建 `nginx.conf`：

```nginx
server {
    listen 80;
    server_name _;

    root /usr/share/nginx/html;
    index index.html;

    location / {
        try_files $uri $uri/ /index.html;
    }

    location /api/ {
        proxy_pass http://dorisoy.pan:80/api/;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        client_max_body_size 500M;
    }

    location /userHub {
        proxy_pass http://dorisoy.pan:80/userHub;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_set_header Host $host;
        proxy_read_timeout 86400s;
    }

    gzip on;
    gzip_types text/plain text/css application/json application/javascript text/xml;
}
```

### 5.3 构建并运行

```bash
cd UI/
docker build -t dorisoy-pan-ui:latest .

docker run -d \
  --name dorisoy-pan-ui \
  --restart always \
  --link dorisoy.pan \
  -p 80:80 \
  dorisoy-pan-ui:latest
```

或者使用 Docker Compose 将前端容器加入后端所在网络：

```yaml
version: "3.8"
services:
  ui:
    build:
      context: ./UI
      dockerfile: Dockerfile
    container_name: dorisoy-pan-ui
    restart: always
    ports:
      - "80:80"
    # 与后端容器在同一网络
    external_links:
      - dorisoy.pan
```

## 七、常见问题

### 7.1 页面空白 / JS 加载失败

- 检查 Nginx `root` 路径是否指向正确的构建产物目录
- 确认 `dist/your-drive/` 下有 `index.html` 文件
- 查看浏览器 DevTools Console 中的错误信息

### 7.2 API 请求 404 或 CORS 错误

- **同域部署**：确认 Nginx 的 `location /api/` 代理配置正确，`proxy_pass` 指向 `http://127.0.0.1:5000/api/`
- **跨域部署**：确认后端 `appsettings.json` 中 `WebApplicationUrl` 包含前端域名，格式示例：`"http://your-domain.com,http://localhost:4200"`
- 检查 `environment.prod.ts` 中的 `apiUrl` 配置是否正确

### 7.3 刷新页面 404

Angular 为单页应用（SPA），所有路由由前端处理。确认 Nginx 配置了：

```nginx
location / {
    try_files $uri $uri/ /index.html;
}
```

### 7.4 SignalR 实时通知不工作

- 确认 Nginx 配置了 `/userHub` 的 WebSocket 代理
- 确认 `proxy_set_header Upgrade` 和 `Connection "upgrade"` 已设置
- 如使用 HTTPS，确保 WebSocket 也走 `wss://`

### 7.5 文件上传失败 / 超时

- 确认 Nginx `client_max_body_size` 足够大（建议 `500M`）
- 确认 `proxy_read_timeout` 和 `proxy_send_timeout` 足够长（建议 `600s`）

### 7.6 构建时 Node.js 版本不兼容

Angular 12 需要 Node.js 14.x 或 16.x。如果使用更高版本，会出现编译错误。请使用 nvm 切换到 Node.js 16：

```bash
nvm install 16
nvm use 16
export NODE_OPTIONS=--openssl-legacy-provider
```

## 八、后端 CORS 配置参考

如果采用前后端分离部署（方式 B），需要确保后端 `appsettings.json` 的 `WebApplicationUrl` 包含前端访问地址：

```json
{
  "WebApplicationUrl": "http://your-domain.com,http://前端IP:端口"
}
```

多个地址用英文逗号分隔。修改后重启后端容器：

```bash
docker restart dorisoy.pan
```
